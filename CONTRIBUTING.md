## 🚀 Getting Started

### 1. Fork the Repository
```bash
# Clone your fork
git clone https://github.com/your-username/ShinuAILabs-MasteryHub.git
cd ShinuAILabs-MasteryHub

# Add upstream remote
git remote add upstream https://github.com/shinuailabs/ShinuAILabs-MasteryHub.git
```

### 2. Pick a Part to Work On
- Browse the [QE-AI-Track](./QE-AI-Track/) or [Playwright-Pro-Track](./Playwright-Pro-Track/)
- Check [open issues](https://github.com/shinuailabs/ShinuAILabs-MasteryHub/issues) for ideas
- Or suggest something new in [Discussions](https://github.com/shinuailabs/ShinuAILabs-MasteryHub/discussions)

### 3. Make Your Changes
```bash
# Create a branch
git checkout -b my-contribution

# Make your edits
# Test everything before committing
```

## 📝 Style Guidelines

### Markdown
- Use ATX headings (`##`, not underlined)
- Include a blank line after headings
- Use fenced code blocks with language tags (```python)
- Tables should be pipe-formatted and readable in raw Markdown

### Code
- Follow the existing style of the Part you're editing
- Include comments in non-obvious sections
- Test all code before committing

### Writing
- **Be clear** — Assume the reader is new to the topic
- **Be concise** — No fluff, get to the point
- **Be practical** — Every section should teach something applicable
- **Use active voice** — "Run this command" not "This command should be run"

## 🔄 Pull Request Process

1. **Before starting**, check if there's already an issue or PR for your idea
2. **Keep PRs focused** — One feature/fix per PR
3. **Test everything** — Make sure code runs, links work, instructions are accurate
4. **Write a clear description** — What changed and why
5. **Add attribution** — If you're adapting content from another source, note it

### PR Title Format
```
type(scope): brief description
```
Examples:
- `fix(part-02): correct RICE-POT example typo`
- `feat(part-08): add Qdrant setup guide`
- `docs(readme): update contributing section`

### Review Process
1. Maintainer reviews within 3-5 business days
2. Address feedback if requested
3. Once approved, your PR will be merged
